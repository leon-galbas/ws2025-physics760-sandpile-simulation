\subsection{Cellular Automata}

Cellular automata (CA) provide a discrete-time framework for modeling the evolution of spatially extended systems with local interactions. A CA is specified by
\begin{itemize}
    \item a lattice (cell space) $R$,
    \item a finite neighbourhood $M \subset R$,
    \item a state space $Q$,
    \item local evolution function $\delta : Q^{M} \to Q$.
\end{itemize}
A configuration at time $t$ is a map $q_t : R \to Q$ that assigns a state to each cell. The dynamics are generated by applying the same local rule to every site. We write the restriction of the configuration to the neighbourhood of $r \in R$ by $q_t|_{r+M}$, the update takes the form
\[
q_{t+1}(r) = \delta\!\bigl(q_t|_{r+M}\bigr).
\]
In this work the lattice $R$ is a $d$-dimensional hypercubic grid of linear size $N$, i.e.\ $|R|=N^d$, together with boundary conditions specified later. Starting from an initial configuration $q_0$, the rule $\delta$ determines the complete time evolution $\{q_t\}_{t\ge 0}$. Many physically motivated CA are additionally driven or perturbed by an external mechanism that modifies the configuration according to a separate prescription (for example, adding or removing state at sites), after which the local relaxation rule is applied again.
