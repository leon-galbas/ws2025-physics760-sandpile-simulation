To reduce the burn-in time, two alternative initialization schemes are tested. In the first scheme, each site is assigned an independent random height in the interval $[0,z_c]$, i.e.\ $z \sim U(0,z_c)$. In the second scheme, the maximally stable configuration is used as the initial state, i.e.\ $z=z_c$ at all sites. The resulting burn-in curves for configuration B are shown in Figures~\ref{fig:N40d2_open_conservative_random_burn_in} and~\ref{fig:N40d2_open_conservative_max_burn_in}. Compared to the zero initialization, both schemes reach the stationary regime significantly faster. For configuration A, the burn-in time does not change appreciably, as it is already close to smalles possible burn in time resulting from the algorithm introduced in the methods section.

Burn-in times for configurations A--F are measured for all three initialization schemes. The corresponding values are reported in Table~\ref{tab:burn_in_times}. For most configurations, the shortest burn-in time is obtained with the maximal initialization. All subsequent simulations and measurements therefore use this initialization scheme.





To extract scaling exponents and the corresponding power-law prefactors, probability distribution functions (PDFs) are computed as described in the previous chapter and evaluated on log-log axes. In such a representation, power-law behaviour appears approximately linear, with the slope corresponding to the scaling exponent. The exponent is therefore estimated by linear regression in the scaling regime. Deviations from linearity are expected both at small values (discreteness and microscopic effects) and at large values (finite-size effects), as discussed in the Methods section, so the regression range must be selected with care.

To determine a suitable fitting window in a reproducible way, a rolling-window procedure is employed. A linear regression is performed on each window, and for the $i$-th window the slope $m_i$, its standard error $\delta m_i$, and the coefficient of determination $r_i^2$ are recorded. Adjacent windows are then compared by forming the slope difference $d_i$ and its propagated uncertainty $\delta d_i$,
\begin{equation}
    d_i = \abs{m_i-m_{i+1}}, \qquad
    \delta d_i = \sqrt{(\delta m_i)^2+(\delta m_{i+1})^2}.
\end{equation}
A window pair is classified as stable if the slope variation is statistically compatible with zero, $d_i < k\,\delta d_i$, and if both fits satisfy $r_i^2 \ge r_{\mathrm{thresh}}$ and $r_{i+1}^2 \ge r_{\mathrm{thresh}}$, where $k$ is a chosen deviation factor and $r_{\mathrm{thresh}}$ is a minimum fit-quality threshold. The final fitting interval is obtained by selecting the longest contiguous stable region and using its corresponding lower and upper boundaries for the exponent regression. At this point it is important to note that the procedure described above is heuristic and does not yield an exact or unique scaling window. The parameters $k$ and $r_{\mathrm{thresh}}$, as well as the window size, are tuned manually and can vary between model configurations.
